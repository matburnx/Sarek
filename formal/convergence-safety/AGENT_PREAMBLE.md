## Project in one paragraph

Formal specification and CMBT certification of `sarek/ppx/Sarek_convergence.ml`,
the GPU barrier-safety analysis pass for the Sarek OCaml GPU DSL (part of SPOC).
The analysis tracks an execution mode (`Converged | Diverged`) through a typed
AST (`texpr`) and collects `BarrierError` when a synchronisation point
(`TEIntrinsicFun _ (Some ConvergencePoint) _`) is reached inside diverged
control flow. The Rocq spec (`theories/ConvergenceSpec.v`) proves 11 theorems
about the abstract `expr` model; the extraction (`extraction/ConvergenceModel.ml`)
is differentially tested against the inline abstract model by
`test/test_convergence_extraction.ml`.

## Hard constraints

### Policy

- Every new `.ml` test executable must call at least one `Sarek_convergence.*`
  or `Sarek_core_primitives.is_thread_varying` symbol in its body.
  `policy/CONFORMANCE_POLICY.md §"What counts as a sarek symbol"`.
- PBT tests use **QCheck2** only (no Alcotest, no bam, no AFL).
- Hand-written regression tests go under `test/regressions/` and MUST link
  to a Finding in `findings/DIVERGENCE_FINDINGS.md`.

### Assertions

- Strict list equality on `check` results. No approximate comparisons.
- Extraction test: `normalize_errors` must be applied before comparing
  `ConvergenceModel.check` output to inline model output.

### Failure handling

When an extraction conformance test in `test/` fails:

- Do NOT install a gate without classification.
- Phase 1 (harness-bug triage) is autonomous — follow `policy/CONFORMANCE_POLICY.md
  §"Conformance test fails — two-phase triage"`.
- Phase 2 (spec-vs-proto classification) is user-gated. Fill the Finding template
  in `findings/DIVERGENCE_FINDINGS.md`, present it, stop.

### Scope discipline

- Do not touch `theories/ConvergenceSpec.v` unless the task is explicitly
  about a spec change.
- Do not touch `extraction/ConvergenceModel.ml` by hand — it is
  auto-generated by `coqc`. Re-run extraction after spec changes.
- Keep each commit to a single concern.

### Reporting

- Update `STATUS.md §"Test intensity"` when test counts change.
- Update `report/BENCHMARKS.md` when eq-class numbers change.
- Append a one-line session entry to `history/JOURNAL.md`.

### LaTeX source-of-truth

Pre-lock: LaTeX (`ConvergenceSafetySpec.tex`) and Rocq co-evolve.
Post-lock (current state): every spec change goes into `ConvergenceSafetySpec.tex`
FIRST; Rocq follows. Pre-lock reconciliation mandatory: diff LaTeX statements
vs Rocq theorem signatures. Drift blocks lock.
